SkeletonCSS 2.1 Changes & Examples

Introduction

SkeletonCSS is an expansion upon the original SkeletonCSS framework designed by Dave Gamache. It includes components from the Skeleton on SASS project by Seth Coelen. I have made my own changes as well, integrating many of the changes I typically make when starting a new project with Skeleton. Many of these changes are subjective and match my personal preferences, which may differ from the preferences of others. Overall my goal is to do small fixes and changes to Skeleton to update it for 2016 and onward, along with altering some of the typographic elements I have always changed in my own projects.

Here is an overview:

  • Ported SkeletonCSS to SCSS using the Skeleton on SASS project.
  • Updated normalize to latest version (4.1.1)
  • Increased container width to 1200px maximum size
  • Added margins to improve readability
  • Switched body font to Roboto
  • Switched body font color to #444 to improve readability
  • Default font size now 1.6em (16px) to improve readability and line-height to 1.46
  • Re-scaled headers
  • Changed list item styles to disc from circle
  • Added new HTML5 input elements

Body & Containers

Width

The maximum container width has been increased from 960px in SkeletonCSS 2.0 to 1200px. This allows pages to span wider screens before maxing out in size.

Margins

A 1.5rem margin has been added to the top of the body to separate headers from the top of the page.

SCSS

Thanks to the Skeleton on SASS project, SkeletonCSS is now broken down into SCSS components, included in this project.

Typography

Color

Small changes have been made to the color of the typography of SkeletonCSS. The <p> text is now slightly lighter in color than the headers, #444 versus #222.

Font Family

Roboto has been added as a body font for Skeleton.

Font Size

Body font sizing has been changed to 1.6rem, or 16px.

Headings have been changed as well to better match in size.
